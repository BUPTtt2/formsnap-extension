## 问题清单

### 问题1 [严重] 坚果云类在线表格表头识别失败

- **入口**: `src/content/formScanner.ts:36-47` → `getLabelForElement()`
- **模块**: 表单扫描模块
- **行为**: 仅检查 `tag === 'th'` 和 `role === 'columnheader'`，但坚果云等在线表格使用 `<td>` 作为表头列名。遍历所有行后找不到匹配，返回空字符串
- **后果**: 所有 contenteditable 单元格 label 为空 → `clientSideMatch` 中 `tgtNorm = ''`，`bigramSimilarity` 返回 0 → 所有字段显示"无对应列"
- **修复**: 在 Strategy 1 末尾添加 fallback：如果没有找到 `<th>`，遍历所有行取第一个非空文本作为表头
- **状态**: ✅ 已修复

### 问题2 [严重] 客户端匹配完全依赖 label

- **入口**: `src/popup/Popup.tsx:323-358` → `clientSideMatch()`
- **模块**: 匹配模块
- **行为**: 仅用 `tgt.label` 和 `tgt.name` 做匹配，当 label 和 name 都为空时，所有匹配失败
- **后果**: 问题1 导致 label 为空后，即使 placeholder、id 有值也无法匹配
- **修复**: 添加 `tgt.placeholder`、`tgt.id` 作为精确匹配和 bigram 相似度匹配的候选字段
- **状态**: ✅ 已修复

### 问题3 [中] 映射历史存储无上限 → 可能耗尽存储

- **入口**: `src/background/index.ts:37-44` → `SAVE_MAPPING_HISTORY`
- **模块**: 后台模块
- **行为**: 每次填写成功都保存映射历史到 chrome.storage.local，无数量限制
- **风险**: 长时间使用可能耗尽 chrome.storage.local 配额（约 10MB）
- **修复**: 限制最多保存 50 条记录，超出时自动删除最旧的
- **状态**: ✅ 已修复

### 问题4 [中] 消息处理缺少参数校验

- **入口**:
  - `src/background/index.ts` → `PARSE_DATA_SOURCE`、`SAVE_SETTINGS`、`GET_MAPPING_HISTORY`
  - `src/content/index.ts` → `SCAN_FORM`、`FILL_FORM`、`CLICK_NEW_BUTTON`
- **模块**: 消息路由模块
- **行为**: 直接使用 `message.payload.xxx` 而不做 null/undefined 检查，payload 可能为 undefined 导致运行时异常
- **风险**: 异常消息导致扩展崩溃
- **修复**: 所有消息处理分支添加 payload 校验和类型检查
- **状态**: ✅ 已修复

### 问题5 [中] 选择器生成不稳定

- **入口**: `src/content/formScanner.ts:95-125` → `getSelector()`
- **模块**: 表单扫描模块
- **行为**: 生成的 CSS 选择器依赖 DOM 类名和结构位置，动态渲染页面可能改变选择器稳定性
- **风险**: 页面刷新或交互后，保存的选择器可能定位到错误元素
- **建议**: 后续可添加多种选择器策略（如 data-* 属性优先、id 优先等）
- **状态**: 📋 已知限制，可后续优化

### 问题6 [低] `fillContentEditable` 破坏富文本结构

- **入口**: `src/content/formFiller.ts:30-42` → `fillContentEditable()`
- **模块**: 表单填充模块
- **行为**: `el.textContent = value` 会清除所有子元素（如 `<br>`、`<span>`、`<div>` 等），对富文本编辑器可能破坏已有结构
- **风险**: 在带格式的 contenteditable 区域中填入纯文本，会丢失原有格式
- **建议**: 后续可改用 `el.innerText = value` 或更精细的 DOM 操作
- **状态**: 📋 已知限制，可后续优化

### 问题7 [低] `host_permissions: ["<all_urls>"]` 权限过大

- **入口**: `public/manifest.json:7`
- **模块**: 权限声明
- **行为**: 扩展声明了访问所有网站的权限
- **风险**: Chrome Web Store 审核时可能被关注；但功能需要（填写任意网页表单），属于合理声明
- **状态**: ✅ 合理使用，无需修改

## 未发现的安全漏洞

经过审查，以下常见安全风险未在代码中发现：

| 检查项 | 结果 | 说明 |
|--------|------|------|
| DOM-based XSS | ✅ 安全 | React 默认防止 XSS，未发现 `dangerouslySetInnerHTML` 或 `innerHTML` 赋值 |
| API Key 存储 | ⚠️ 已知限制 | 存储在 chrome.storage.local，Chrome 扩展标准做法 |
| 截图数据泄露 | ⚠️ 已知限制 | IndexedDB 存储未加密，但仅在本地扩展上下文中可访问 |
| CSRF 防护 | ✅ 不适用 | 纯前端扩展，无后端接口 |
| 权限提升 | ✅ 安全 | 最小权限集：activeTab + scripting + storage |
| 内容脚本注入 | ✅ 安全 | 使用 `chrome.scripting.executeScript` 注入自身 bundle，无动态代码执行 |

## 异常场景总结

当前已覆盖的异常场景（含本次修复后）：

| 场景 | 处理方式 | 状态 |
|------|---------|------|
| API Key 未配置 | Popup 提示"请设置 API Key" | ✅ |
| API 401/403/429/500/503 | 友好错误提示，区分具体原因 | ✅ |
| AI 返回空内容 | throw Error("AI 返回了空内容") | ✅ |
| AI JSON 解析失败 | 平衡括号解析 + 尾逗号修复 + 单引号修复 | ✅ |
| 截图超过大小限制 | 提示"图片超过 5MB 限制" | ✅ |
| 截图数量超过上限 | 提示"最多支持 10 张截图" | ✅ |
| 文件超过大小限制 | 提示"文件超过 10MB 限制" | ✅ |
| 内容脚本未注入 | 动态注入 content.js 并重试 | ✅ |
| 页面无法连接 | 提示"刷新页面后重试" | ✅ |
| 未检测到表单字段 | 提示"检测到新建按钮"或"未检测到表单字段" | ✅ |
| 点击新建后仍未加载表单 | 提示"需再次点击"或"手动操作后重试" | ✅ |
| 无可用映射 | 提示"没有可填写的映射" | ✅ |
| 数据源未识别到字段 | 提示"未能从数据源中识别到任何字段" | ✅ |
| 坚果云类 `<td>` 表头 | Fallback 到第一行文本作为表头 | ✅ **本次新增** |
| 匹配时 label 为空 | Fallback 到 placeholder、id | ✅ **本次新增** |
| 映射历史存储超限 | 自动截断至 50 条 | ✅ **本次新增** |
| 消息 payload 为空 | 参数校验，返回友好错误 | ✅ **本次新增** |
| 状态恢复超时 | 30 分钟过期自动清除 | ✅ |
| 填写撤销 | undoStack 记录原始值 | ✅ |
