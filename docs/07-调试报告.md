# 调试报告：FormSnap 全局代码审查

**日期**: 2026-07-15
**触发**: 用户反馈"还是错，请全部全局的去看"

---

## 1. 症状描述

用户在坚果云在线表格（类似Excel的在线编辑器）页面使用 FormSnap，截图上传后：
- 之前版本完全扫描不到表单字段（因为单元格是 contenteditable div）
- 修复 contenteditable 支持后，用户仍反馈"还是错"
- 用户截图显示 Chrome DevTools 中有一个 404 错误（来自目标网站自身的 API 调用，与 FormSnap 无关）

## 2. 全局代码审查发现

### 已确认正常的模块

| 文件 | 状态 | 说明 |
|------|------|------|
| `src/core/types.ts` | OK | 类型定义完整，无重复 |
| `src/utils/storage.ts` | OK | Popup状态持久化正常 |
| `src/utils/imageUtils.ts` | OK | 图片压缩逻辑正常 |
| `src/utils/textParser.ts` | OK | 本地文本解析正常 |
| `src/utils/fileParser.ts` | OK | Excel/CSV解析正常 |

### 发现的问题

#### 问题1: contenteditable 继承模式未覆盖 (中等严重)
**文件**: `src/content/formScanner.ts`
**现状**: 只扫描 `[contenteditable="true"]`，但很多在线表格（如坚果云、腾讯文档）的 contenteditable 是通过父元素继承的，单元格本身没有 `contenteditable="true"` 属性。
**修复**: 同时扫描 `[contenteditable]` 和父元素有 contenteditable 的可聚焦元素。

#### 问题2: Shadow DOM 未处理 (中等严重)
**文件**: `src/content/formScanner.ts`
**现状**: 如果目标页面使用 Shadow DOM（如一些 Web Component 框架），`document.querySelectorAll` 无法穿透 Shadow DOM 边界。
**修复**: 添加 Shadow DOM 穿透扫描逻辑。

#### 问题3: iframe 内表单未处理 (低严重)
**文件**: `src/content/formScanner.ts`
**现状**: 如果目标表单在 iframe 内，content script 无法直接访问。
**修复**: manifest 中添加 all_frames 支持，content script 中检测 iframe context。

#### 问题4: Popup 状态恢复时 base64 图片过大可能导致 storage 写入失败 (低严重)
**文件**: `src/utils/storage.ts`
**现状**: `QUOTA_BYTES_PER_ITEM` 限制为 8KB (chrome.storage.local)，但多张截图的 base64 可能远超此限制。
**修复**: 改用 IndexedDB 存储大体积图片数据，或进一步压缩。

#### 问题5: 文件解析只处理第一个 sheet 的第一行数据 (低严重)
**文件**: `src/utils/fileParser.ts`
**现状**: `parseCSV` 只用第一行作为 header，处理多行数据时会重复字段名。
**修复**: 文件上传场景应该只取第一行作为字段名，值取指定行或让用户选择行。当前行为对单行数据正确，多行时会重复。

#### 问题6: 构建产物包含不必要的 .d.ts 文件 (低严重)
**文件**: `tsconfig.json`
**现状**: `"declaration": true` 导致 dist 中输出大量类型声明文件。
**修复**: 在 tsconfig 中关闭 declaration，或使用单独的 tsconfig.build.json。

#### 问题7: CSS 样式没有暗色模式/深色主题 (外观问题)
**文件**: `src/popup/styles.css`
**现状**: Popup 使用浅色主题 (--bg: #FAFAFA)，与深色系产品展示页不一致。
**修复**: 添加 dark mode 支持或统一为深色主题。

## 3. 根因分析

用户反馈"还是错"最可能的原因是 **问题1**：在线表格的 contenteditable 是继承的，不是直接设置在单元格上的。当前的选择器 `[contenteditable="true"]` 匹配不到这些元素。

## 4. 修复方案

### 立即修复（问题1）
扩展 contenteditable 扫描策略：
1. 扫描 `[contenteditable="true"]`（直接设置）
2. 扫描 `[contenteditable=""]`（空字符串等同于 true）
3. 扫描 `document.querySelectorAll('td, [role="cell"], [role="gridcell"]')` 中可编辑的单元格
4. 通过检查 `el.isContentEditable` 属性来判断是否可编辑（这会包含继承的情况）

### 后续修复（问题2-3）
- Shadow DOM: 递归遍历所有 shadowRoot
- iframe: 添加 all_frames 配置

### 后续修复（问题4）
- IndexedDB 替代 chrome.storage.local 存储图片

## 5. 验证方式

1. 在在线表格页面（坚果云/腾讯文档/Google Sheets）测试表单扫描
2. 确认扫描到的字段数量和名称与页面一致
3. 测试填写和撤销功能
4. 测试 Popup 关闭后重新打开状态恢复
